Claude Code Balanced Statusline

把项目、分支、上下文、变更量、模型和 effort 档位压进一条高密度状态行。 足够克制，足够醒目，也足够适合一直挂在终端底部。

开始安装 查看信息结构
Ctx 3% context load
+213 / -47 session diff
[max] effort level
zsh - claude-code-balanced-statusline
~/code/my-project $ claude
Loaded project settings from ~/.claude/settings.json
Status line command attached.
claude-code-balanced-statusline   [feature/v1.1.0]*   Ctx: 3%   +213/-47   deepseek-v4-pro   [max]

一眼扫完该看的信息

默认状态栏太空，复杂面板又太吵。Balanced Statusline 只留下会影响当前编码判断的信号。

目录 claude-code-balanced-statusline

始终知道当前会话落在哪个项目里。

Git 分支 [feature/v1.1.0]*

* 表示工作区存在未提交修改。

上下文用量 Ctx: 3%

低用量青色，中段黄色，高压红色。

会话变更 +213/-47

新增和删除行数分色展示。

模型 deepseek-v4-pro

自动去掉 [1m] 这类尾缀。

Effort [max]

medium 保持静默，其他档位才显示。

颜色只服务语义

没有随机装饰色，每个高亮都对应状态栏里真实可判断的信息。

项目目录

当前路径的锚点。

Git / 新增

分支和新增行数。

低上下文

< 50% 保持清爽。

高压 / 删除

>= 80% 和删除行数。

设 NO_COLOR=1 或 CLAUDE_STATUSLINE_COLOR=0 可关闭颜色输出。

安装就一行命令

安装器会自动备份旧文件，只追加状态栏配置，不覆盖你已有的 settings.json。

curl -fsSL https://raw.githubusercontent.com/qipeijun/claude-code-balanced-statusline/main/install.sh | bash

前提：brew install jq，或使用系统对应的包管理器安装 jq。Git 可选。

powershell -c "irm https://raw.githubusercontent.com/qipeijun/claude-code-balanced-statusline/main/install.ps1 | iex"

前提：jq 和 git。没有的话先跑 winget install jqlang.jq Git.Git。

手动安装

# 复制脚本
mkdir -p ~/.claude
cp statusline.sh ~/.claude/statusline.sh
chmod +x ~/.claude/statusline.sh

# 然后在 ~/.claude/settings.json 里加上：
{
  "statusLine": {
    "type": "command",
    "command": "~/.claude/statusline.sh"
  }
}

本地测试

用样例 JSON 直接喂给脚本，确认输出是否符合你当前终端的显示效果。

# 用样例数据跑一下
cat examples/sample-input.json | ./statusline.sh

# 去掉颜色看纯文本
cat examples/sample-input.json | ./statusline.sh | perl -pe 's/\e\[[0-9;]*m//g'
可选显示 user@host 
export CLAUDE_STATUSLINE_SHOW_HOST=1
