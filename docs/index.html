这是什么

Claude Code 的默认状态栏能看，但信息密度为零。Balanced Statusline 用 200 行 Bash 把当前项目、Git 分支、上下文用量、会话变更量、模型名和 effort 档位 压成一行——刚好一眼扫完。

不依赖 Node，不拉一堆包。一个 jq，一个 git（可选），剩下的都是 标准 POSIX。

一行里有什么

目录 claude-code-balanced-statusline
Git 分支 [feature/v1.1.0]*

* 表示有未提交的修改

上下文用量 Ctx: 3%

< 50%  50–79%  ≥ 80%

变更量 +213/-47
模型 deepseek-v4-pro

自动去掉 [1m] 等后缀

Effort [max]

medium 时不显示

配色规则

没有随机渐变色。每个颜色对应一种信息语义：

项目目录
Git 分支
上下文 < 50%
上下文 50–79%
上下文 ≥ 80% & 删除行
新增行
标签 / 模型 / effort

设 NO_COLOR=1 或 CLAUDE_STATUSLINE_COLOR=0 关闭所有颜色。

安装

一行命令，自动备份旧文件，只追加配置字段，不动你已有的 settings.json。

curl -fsSL https://raw.githubusercontent.com/qipeijun/claude-code-balanced-statusline/main/install.sh | bash

前提：brew install jq（或对应系统的包管理器）。Git 可选。

powershell -c "irm https://raw.githubusercontent.com/qipeijun/claude-code-balanced-statusline/main/install.ps1 | iex"

前提：jq 和 git（Git Bash 自带即可）。 没有的话 winget install jqlang.jq Git.Git。
如果 PowerShell 报执行策略错误，先跑：Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

安装器会在 ~/.claude/ 下操作。如果该目录不存在会自动创建；权限不够会明确报错并退出。

手动安装

# 复制脚本
mkdir -p ~/.claude
cp statusline.sh ~/.claude/statusline.sh
chmod +x ~/.claude/statusline.sh

# 然后在 ~/.claude/settings.json 里加上：
{
  "statusLine": {
    "type":    "command",
    "command": "~/.claude/statusline.sh"
  }
}

可选：显示 user@host

export CLAUDE_STATUSLINE_SHOW_HOST=1

多机器共用配置文件时比较有用。

本地测试

# 用样例数据跑一下
cat examples/sample-input.json | ./statusline.sh

# 去掉颜色看纯文本
cat examples/sample-input.json | ./statusline.sh | perl -pe 's/\e\[[0-9;]*m//g'
